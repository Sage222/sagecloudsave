Setup Guide — Ludusavi + rclone SMB

      


        

          
          
Why did backups show "success" but nothing appeared on your share?

          The previous version of this dashboard faked all data — pre-seeded game lists, fake log entries, fake "Backed up" statuses, fake file transfers. None of it was real. This version (v3) shows no fake data. It only generates commands for you to run.

        


        


          

            
1

            
Install rclone

            
Download from rclone.org/downloads. Extract rclone.exe to a folder on your PATH (e.g. C:\tools\rclone\). Verify with:

            
rclone version

          


          

            
2

            
Create the SMB remote in rclone

            
Run interactively. Name it nas (or update SMB Config to match whatever name you choose).

            
rclone config
# → n (new remote)
# → name: nas
# → type: smb
# → host: 192.168.1.50
# → user: sage
# → pass: (enter when prompted, or leave blank for guest)
# → leave all other fields blank → q to quit

          


          

            
3

            
Verify rclone can see the share

            
This is the most important test. If this fails, nothing else will work.

            

            
Expected output: a list of folders on the share, or an empty result (not an error). If you get an error, fix the host/credentials before continuing.

          


          

            
4

            
Set rclone path in Ludusavi

            
In Ludusavi: Other tab → Rclone executable → point it to rclone.exe. Then: Other → Cloud → Add → SMB → enter your remote name (e.g. nas) and path (e.g. GameSaves/ludusavi). Turn off auto-sync.

          


          

            
5

            
Understand the two-step backup process

            
Critical: ludusavi backup only copies saves to a local staging folder on your PC. The files do NOT go to the SMB share yet. You must then run ludusavi cloud upload as a separate step to push them.

            

          


          

            
6

            
Troubleshooting: nothing on share

            
Work through these in order:

            

              
①Did you run cloud upload? Backup alone does nothing on the share.

              
②Test rclone directly: rclone lsd nas: — must return a listing, not an error.

              
③Check Ludusavi's rclone path: Other tab → Rclone executable must point to the actual rclone.exe binary.

              
④Check remote name: Run rclone listremotes — the name shown must match what Ludusavi has configured.

              
⑤Check the subfolder exists or that rclone has write permission to create it.

              
⑥Run with verbose output: ludusavi cloud upload --verbose — this shows exactly what rclone is doing and any errors it returns.

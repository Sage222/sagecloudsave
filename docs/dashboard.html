Setup Guide — SMB + Ludusavi

      


        
Good news: Your fork (Ludusavi v0.31.0) natively supports SMB shares via rclone — no Google Drive or internet required. Just a NAS, PC, or any Windows file share on your network.


        


          

            
1

            

              
Install rclone

              
Download rclone.exe from rclone.org/downloads and place it anywhere (e.g. C:\tools\rclone\). You don't need to configure rclone manually — Ludusavi has a built-in SMB setup wizard.

            

          


          

            
2

            

              
Configure SMB in Ludusavi GUI

              
Open Ludusavi → Other tab → Cloud section → click Add. Choose SMB server, enter your share host, share name, username and password. Ludusavi calls rclone to validate the connection.

            

          


          

            
3

            

              
Or configure via rclone CLI (one-time)

              
Alternatively run rclone config, choose New remote → SMB, name it nas. Enter host, share, user, password. This creates an entry in rclone.conf that Ludusavi reads automatically.

              
[nas]
type = smb
host = 192.168.1.50
user = sage
pass = <rclone-obscured-password>
# No "share" field needed — set it in Ludusavi's cloud path

            

          


          

            
4

            

              
Set cloud path in Ludusavi

              
In Ludusavi's Other → Cloud, set the remote path to something like GameSaves/ludusavi. This becomes the folder on your share where all game backups are stored.

            

          


          

            
5

            

              
Selectively backup/restore

              
Use this dashboard to pick games by checkbox and click Backup or Restore. Under the hood it runs:
ludusavi backup --force "Game Name" then ludusavi cloud upload — or reverse for restore. No auto-sync, only when you click.

            

          


          

            
6

            

              
Two-PC workflow (no conflicts)

              
Since you trigger sync manually, there's no race condition. Finish playing on PC-1 → click Backup. Sit at PC-2 → click Restore. The SMB share is the single source of truth. Use Ludusavi's Redirects (Other tab) to handle different paths between Windows installs.

            

          


        


        

          
Resulting config.yaml (cloud section)

          
cloud:
  remote:
    type: Smb            # native Ludusavi SMB type via rclone
    id: nas              # your rclone remote name
  path: "GameSaves/ludusavi"
  synchronize: false     # manual only — you control when it runs

Setup Guide — SMB + Ludusavi

      

        
Dashboard is UI-only. This dashboard visualises operations but does not directly invoke Ludusavi or rclone. Backups shown as "success" here simulate the workflow — real file transfers happen through Ludusavi's GUI or CLI. See Step 5 for how to wire them together.


        

          

            
1

            
Install rclone
Download rclone.exe from rclone.org/downloads. Place it at C:\tools\rclone\rclone.exe or anywhere on your PATH.

          

          

            
2

            
Configure SMB remote in rclone
Run once in a terminal. Or use Ludusavi's built-in wizard: Other → Cloud → Add → SMB server.

            
rclone config
# → New remote → name it "nas" → type: smb
# → host: 192.168.1.50
# → user: sage
# → password: (enter when prompted)
# → leave other fields blank → quit

          

          

            
3

            
Point Ludusavi at the rclone remote
In Ludusavi GUI: Other tab → Cloud section → select remote nas, set path to GameSaves/ludusavi. Set rclone executable path. Turn off auto-sync (manual only).

            
# config.yaml (cloud section)
cloud:
  remote:
    type: Smb
    id: nas
  path: "GameSaves/ludusavi"
  synchronize: false   # ← manual only, never auto

          

          

            
4

            
Verify rclone can reach the share
Test before relying on it — rclone will fail silently if credentials are wrong or the share is unreachable.

            
# Should list files/folders on the share:
rclone ls nas:GameSaves/ludusavi

# If empty share, just check it mounts:
rclone lsd nas:

          

          

            
5

            
Manual backup / restore workflow
Use Ludusavi GUI (three-dot menu per game → Backup/Restore), or CLI for scripting. The cloud upload/download step is what actually moves files to/from the share.

            
# BACKUP a game to local staging, then push to SMB share:
ludusavi backup --force "Elden Ring"
ludusavi cloud upload

# RESTORE from SMB share to local saves:
ludusavi cloud download
ludusavi restore --force "Elden Ring"

# Check what would sync (dry run):
ludusavi backup "Elden Ring"    # no --force = preview only

# See verbose rclone output for debugging:
ludusavi cloud upload --verbose

          

          

            
6

            
Debugging: nothing appears on share

            
Common causes — check each:

            

              
①cloud upload not run — ludusavi backup only writes to a local staging folder (~/ludusavi-backup/). You must also run ludusavi cloud upload to push to the share.

              
②wrong rclone path — verify Ludusavi's rclone executable setting points to the actual binary. Run rclone version at that path to confirm.

              
③remote not configured — run rclone listremotes to confirm nas: exists. If not, re-run rclone config.

              
④path mismatch — the path in Ludusavi's cloud config must match a folder that actually exists on the share, or rclone must have permission to create it.

              
⑤game not scanned — Ludusavi must detect the game's save path first. Do a full scan from the Backup tab, then use the per-game menu.
